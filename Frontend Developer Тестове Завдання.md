Необхідно реалізувати невеликий веб\-додаток — спрощену систему подачі та обробки заявок (Mini Request System).

У додатку повинна бути можливість перемикання ролі користувача:

- User (звичайний користувач)
- Manager (менеджер)

Залежно від обраної ролі інтерфейс та доступний функціонал повинні змінюватися.

### **Функціональні вимоги**

#### **Роль User**

Користувач має можливість створювати заявки та переглядати власний список.

Функціонал:

- Форма створення заявки:
  - поле Title
  - поле Description
- Після створення заявка додається у список зі статусом "new"
- Відображення списку заявок:
  - назва
  - короткий опис
  - статус

#### **Роль Manager**

Менеджер має можливість переглядати всі заявки та змінювати їх статус.

Функціонал:

- Список усіх заявок:
  - ID
  - Title
  - Description
  - Status
- Можливість змінювати статус заявки:
  - new → in progress
  - in progress → done
- Фільтрація заявок за статусом:
  - всі
  - new
  - in progress
  - done

### **Хостинг готового ТЗ на Vercel**

Після виконання тестового завдання його необхідно розгорнути на [Vercel](https://vercel.com/) (доступний безкоштовний тариф). За потреби, можна використати власне рішення для хостингу.

- Реалізація перемикання ролей

Додатково (не є обов’язковим):

- Використання Redux
- Використання TypeScript
- Валідація форми
